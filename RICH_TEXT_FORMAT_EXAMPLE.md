# 富文本模式格式说明

## 新建协议时的格式

### 1. 简单文本格式（推荐）

在富文本编辑器中，你可以直接输入文本，系统会自动转换为 HTML。

**示例输入：**
```
付费服务协议

更新时间：2024年07月09日
生效日期：2024年07月24日

尊敬的用户：

欢迎您使用付费服务...
```

**保存后会自动转换为：**
```html
<p>付费服务协议</p>
<p>更新时间：2024年07月09日</p>
<p>生效日期：2024年07月24日</p>
<p>尊敬的用户：</p>
<p>欢迎您使用付费服务...</p>
```

### 2. HTML 片段格式

你也可以直接粘贴 HTML 代码片段，系统会保存为 HTML。

**示例输入（粘贴到富文本编辑器）：**
```html
<p class="p-tag" style="text-align: center;">付费服务协议</p>
<p class="p-tag" style="text-indent: 21pt;">
  欢迎您使用<span class="appName"></span>付费服务。
</p>
<b class="b-tag">重要提示</b>
```

**保存后格式：**
```html
<p class="p-tag" style="text-align: center;">
  付费服务协议
</p>
<p class="p-tag" style="text-indent: 21pt;">
  欢迎您使用
  <span class="appName"></span>
  付费服务。
</p>
<b class="b-tag">
  重要提示
</b>
```

### 3. 完整 HTML 文档格式

如果你粘贴完整的 HTML 文档（包含 `<html>`, `<head>`, `<body>` 标签），系统会：

- 提取 `<body>` 内容显示在编辑器中
- 保存时保留完整的 HTML 结构

**示例输入（完整文档）：**
```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>协议标题</title>
</head>
<body>
  <p>协议内容</p>
</body>
</html>
```

**保存后：**
- 保留 `<!DOCTYPE html>`
- 保留 `<head>` 内容
- 保留 `<body>` 标签属性（如果有）
- 保留编辑后的 body 内容

## 实际使用示例

### 示例 1：简单的协议内容

**在富文本编辑器中输入：**
```
长江云短剧付费服务协议

更新时间：2024年07月09日

一、协议条款的确认和接受

1.1 用户在使用本服务前，需要仔细阅读并同意本协议的全部内容。
1.2 用户点击"同意"按钮，即表示用户已充分理解并同意本协议的全部内容。
```

**保存后的 HTML（格式化后）：**
```html
<p>
  长江云短剧付费服务协议
</p>
<p>
  更新时间：2024年07月09日
</p>
<p>
  一、协议条款的确认和接受
</p>
<p>
  1.1 用户在使用本服务前，需要仔细阅读并同意本协议的全部内容。
</p>
<p>
  1.2 用户点击"同意"按钮，即表示用户已充分理解并同意本协议的全部内容。
</p>
```

### 示例 2：带样式和类的协议内容

**在富文本编辑器中输入（或粘贴 HTML）：**
```html

  欢迎您使用

</p>
<p class="p-tag">
  <b class="b-tag">重要提示：</b>
```

**保存后的 HTML（格式化后）：**

## 注意事项

1. **格式标准化**：浏览器在保存时会自动标准化格式（换行、空白等），这是无法避免的
2. **标签保留**：所有 HTML 标签、CSS 类和内联样式都会被保留
3. **自动格式化**：保存时会自动格式化 HTML，添加缩进和换行，便于阅读
4. **新建 vs 编辑**：
   - 新建协议：直接使用编辑器的内容，不包装成完整 HTML 文档
   - 编辑协议：如果是完整 HTML 文档，会保留完整的结构

## 推荐做法

### 新建协议时：
1. **简单内容**：直接在富文本编辑器中输入文本，让系统自动转换
2. **复杂格式**：可以先在代码模式下写好 HTML，然后切换到富文本模式查看效果
3. **从 Word 复制**：可以直接从 Word 复制内容粘贴到富文本编辑器（格式可能会简化）

### 编辑现有协议时：
1. **富文本模式**：适合修改文字内容，格式可能会被标准化
2. **代码模式**：适合精确控制格式，完全保持原样
